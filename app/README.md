# 農務小幫手 App（PWA）

信義鄉高冷地溫室的 App，**手機可安裝到主畫面、離線使用；電腦瀏覽器打開即是桌面版**。
純前端、無需伺服器、無需帳號，資料存在你自己的瀏覽器（localStorage）。

> **響應式**：同一份 App，手機是底部分頁的手機版；螢幕寬度 ≥900px（電腦/平板橫向）
> 自動切換成**左側選單、寬版面、置中彈窗**的桌面版。功能完全相同。

## 功能

| 分頁 | 功能 |
|---|---|
| 📊 總覽 | KPI（田區數、今日任務、近採收、用藥管制）、各田區**生育進度條**、30 天內即將採收清單 |
| 🌱 田區 | 田區列表 →**點入詳情**：今日待辦、快速登錄紀錄、EC 趨勢圖、紀錄歷史、採收累計 |
| 🔍 診斷 | 症狀 → 可能原因與處置（缺素、生理障礙、病蟲害）互動查詢 |
| 🧮 工具 | 作物計算（採收/肥料/排程）＋ 年度輪種規劃（卡高價＋輪作）|

### 田區管理（核心）
點任一田區進入詳情，可**快速登錄六種紀錄**：施肥 💊、用藥 🧴、採收 🧺、EC/pH 🌡️、作業 ✂️、觀察 👀。

- **用藥紀錄**可填安全採收期天數 → 系統**自動鎖定、在總覽與詳情警示「禁止採收至 X 日」**
- **採收紀錄**自動**累計總產量（kg）**
- **EC/pH 紀錄**自動繪成趨勢圖，掌握鹽分累積
- 每筆紀錄含日期、類型標籤、內容，可刪除

作物知識庫（6 種：牛番茄、彩椒、玉女番茄、高山草莓、高麗菜、敏豆）與 `scripts/crops.json` 同源，數值為示範值，請經**台中區農業改良場**校正。

## 檔案

- `index.html` — App 本體（自帶樣式與邏輯，單檔可獨立運作）
- `manifest.webmanifest` — PWA 設定（名稱、圖示、主題色）
- `sw.js` — Service Worker（離線快取）
- `icon.svg` — App 圖示

## 怎麼用

### 最簡單：直接開
把 `index.html` 用手機瀏覽器開啟即可使用（田區/待辦/計算/規劃都能用）。

### 安裝成 App（需經 https 或 localhost）
PWA 的安裝與離線功能需要透過網頁伺服器（https 或 localhost）提供，不能用 `file://` 直接開。

本機測試：
```bash
cd app
python3 -m http.server 8000
# 手機與電腦同網段時，用手機瀏覽器開 http://<電腦IP>:8000
```

免費上線（擇一）：
- **Netlify（建議）**：連結整個 repo 部署。根目錄 `netlify.toml` 已設定：靜態站 = `app/`，
  並附 **AMIS 行情代理函式** `netlify/functions/amis.js`。部署後「工具 → 市場行情比價」
  即可即時抓各批發市場上/中/下價，判斷上貨/下貨送哪個市場。
- **GitHub Pages / Cloudflare Pages**：也可只放 `app/`，但**無 serverless 函式**，
  市場行情比價需另接代理（見下）。

### 市場行情比價（送哪個市場）

「工具」分頁的「市場行情比價」會 `POST /.netlify/functions/amis`，由 Netlify 伺服器端
代理農業部 AMIS 開放資料（繞過瀏覽器跨域、借用開放網路）。輸入作物與天數，回傳各市場
交易量加權的**上/中/下/均價**，並標出：**上貨送上價最高的市場、下貨送下價最高的市場**。

> 為什麼要 serverless 代理：`data.moa.gov.tw` 不支援瀏覽器跨域直連，且部分開發/雲端環境
> 對外連有政策限制。把抓取放到伺服器端函式即可解決（與 group-meal 專案同一模式）。

開啟後：
- iPhone Safari：分享 → 加入主畫面
- Android Chrome：選單 → 安裝應用程式 / 加到主畫面

安裝後有 App 圖示、全螢幕、離線可用。

## 與 scripts/ 工具的關係

`app/` 是給**現場每天用**的手機介面；`scripts/` 是給**電腦端**做資料抓取與批次分析（如 `fetch_amis.py` 撈真實行情）。兩者共用同一套作物知識與計算邏輯。
