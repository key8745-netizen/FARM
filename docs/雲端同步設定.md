# 雲端同步設定（Firebase / Firestore）

> 目的：讓手機與電腦共用同一份資料（田區、紀錄、契約、行情校正、常送市場）。
> 選用功能——不設定也能用，只是各裝置資料獨立。

## 運作方式

App 把你的全部資料存成雲端一份文件：`farms/{同步碼}`。
- **上傳**：本機 → 雲端（覆蓋雲端）
- **下載**：雲端 → 本機（覆蓋本機）
- **自動同步**：每 15 秒偵測本機有變就上傳；App 開啟時先下載最新。

> ⚠️ 採「最後寫入覆蓋」。同一時間兩台都在改離線資料時，後上傳的會蓋前一份。
> 平常一台記錄、另一台看，不會有問題。

## 一、建立 / 取得 Firebase 設定

1. 到 https://console.firebase.google.com → 建立專案（或用你 group-meal 的專案亦可，資料互不干擾）。
2. 左側 **建構 → Firestore Database → 建立資料庫**（正式版或測試版皆可，規則見下）。
3. 專案設定（齒輪）→ **一般 → 你的應用程式 → 網頁應用程式**（沒有就新增一個網頁 App）。
4. 找到 **SDK 設定與設定 → Config**，複製 `firebaseConfig` 物件，例如：
   ```js
   const firebaseConfig = {
     apiKey: "AIza…",
     authDomain: "your-project.firebaseapp.com",
     projectId: "your-project",
     storageBucket: "your-project.appspot.com",
     messagingSenderId: "1234567890",
     appId: "1:1234:web:abcd"
   };
   ```
5. 在 App「工具 → ☁️ 雲端同步」把**大括號 `{ … }` 那段**貼進去（前面的 `const firebaseConfig =` 可留可不留，App 會自動擷取）。

## 二、設定同步碼

- 自己取一個**長且不易猜**的字串，例如 `xinyi-farm-9x7k2vQ`。
- 兩台裝置填**一樣**的同步碼，就會共用同一份資料。
- 同步碼等於資料鑰匙，**請勿外流**。

## 三、Firestore 安全規則（重要）

預設測試版規則 30 天後會鎖。請到 **Firestore → 規則** 設定。最簡單（單人自用、靠同步碼保護）：

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /farms/{code} {
      allow read, write: if true;   // 靠不易猜的同步碼保護
    }
  }
}
```

> 這代表「知道同步碼就能存取」。因無登入機制，安全性取決於同步碼夠不夠難猜——請設長一點。
> 想更嚴謹可改用 Firebase 匿名登入 + 以 uid 命名文件（需另外開發），或限制到你自己的 uid。

## 四、開始用

1. 手機：貼設定、填同步碼 → **儲存設定** → **⬆ 上傳到雲端**。
2. 電腦：貼**相同**設定與同步碼 → **儲存設定** → **⬇ 從雲端下載**。
3. 兩台都勾**自動同步**，之後改動會自動互通（開啟時拉取、有變更時推送）。

## 五、常見問題

**Q：一定要用 Firebase 嗎？**
A：不用。不設定就各裝置獨立，改用「資料備份 / 還原」JSON 手動搬也可以。

**Q：會不會很花錢？**
A：個人用量極小，Firestore 免費額度（每日 5萬讀、2萬寫）遠遠用不完。

**Q：資料安全嗎？**
A：資料在你自己的 Firebase 專案裡。安全性主要靠同步碼與 Firestore 規則，請設難猜的碼、正式規則勿長期開全放。

**Q：離線能用嗎？**
A：能。App 本身離線可用（資料在本機），同步只在有網路時進行。
