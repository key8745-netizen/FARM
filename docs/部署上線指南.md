# 部署上線指南（Netlify）

> 目的：把「農務小幫手」PWA 部署到 Netlify，取得 https 網址、可裝到手機主畫面、
> 並讓**市場行情比價**能即時抓 AMIS 資料（透過 serverless 代理函式）。
> 全程免費、不需信用卡。約 10 分鐘。

---

## 為什麼要用 Netlify（而不是只丟 GitHub Pages）

| 需求 | GitHub Pages | Netlify |
|---|---|---|
| PWA 安裝、離線 | ✅ | ✅ |
| 田區/待辦/契作/計算 | ✅ | ✅ |
| **即時抓市場行情** | ❌（無伺服器函式） | ✅（`/.netlify/functions/amis`） |

市場行情要打農業部 `data.moa.gov.tw`，瀏覽器**不能跨域直連**，一定要有伺服器端代理。
Netlify 的 serverless 函式就是這個代理（跟你 group-meal 專案同一套做法）。

---

## 一、前置：專案結構（已經幫你設定好）

```
FARM/
├─ netlify.toml              ← 部署設定（靜態站=app/、函式目錄）
├─ app/                      ← PWA 本體（index.html、manifest、sw.js、icon）
└─ netlify/functions/
   └─ amis.js                ← AMIS 行情代理函式
```

`netlify.toml` 內容重點：
```toml
[build]
  publish = "app"            # 靜態網站根目錄
[functions]
  directory = "netlify/functions"
```

你**不用改任何檔案**，直接照下面步驟連上去即可。

---

## 二、部署步驟（用 GitHub 連結，最省事）

1. 到 **https://app.netlify.com** → 用 GitHub 帳號登入。
2. 點 **Add new site → Import an existing project**。
3. 選 **Deploy with GitHub** → 授權 → 選你的 repo **`key8745-netizen/FARM`**。
4. 分支選 **`claude/farm-assistant-data-collection-ugs3wy`**
   （之後合併到 main，就改用 main）。
5. 建置設定 Netlify 會自動讀 `netlify.toml`，通常不用改：
   - Base directory：留空
   - Build command：留空
   - Publish directory：`app`
   - Functions directory：`netlify/functions`（自動偵測）
6. 按 **Deploy site**。等 1~2 分鐘出現綠色 **Published**。
7. 你會拿到一個網址，像 `https://random-name-1234.netlify.app`
   （可在 Site settings → Change site name 改成好記的，例如 `xinyi-farm`）。

---

## 三、確認函式有跑起來

1. 打開你的網址 → 進 **🧮 工具 → 市場行情比價**。
2. 選作物（例如番茄）、近 7 天 → 按 **抓取市場行情**。
3. 正常會出現各市場上/中/下價表；若當天/近日無交易，換「近 30 天」再試。

**若顯示「抓取失敗」**，依序檢查：
- 網址是 `https://...netlify.app` 而**不是** `file://` 或 `localhost`（函式只在 Netlify 上跑）。
- Netlify 後台 **Functions** 分頁應看到 `amis` 這支函式；點進去看 **Logs**。
- Logs 若出現連 `data.moa.gov.tw` 逾時：AMIS 偶爾較慢，重試或縮短天數。
- 確認部署的分支就是含 `netlify/` 資料夾的那個分支。

> 小工具：直接測函式（把網址換成你的）
> 用手機/電腦瀏覽器的開發者工具 Console 貼：
> ```js
> fetch('/.netlify/functions/amis',{method:'POST',headers:{'Content-Type':'application/json'},
>   body:JSON.stringify({crop:'番茄',start:'2026-07-15',end:'2026-07-21'})}).then(r=>r.json()).then(console.log)
> ```
> 有回傳 `markets` 陣列就代表函式正常。

---

## 四、裝到手機主畫面（PWA）

開你的 Netlify 網址後：
- **iPhone Safari**：分享鈕 → 加入主畫面。
- **Android Chrome**：選單 → 安裝應用程式／加到主畫面。

裝好後有 App 圖示、全螢幕、離線可用（行情比價需連線）。

---

## 五、之後更新內容

Netlify 會**自動部署**：你（或我）push 到所連的分支，Netlify 幾分鐘後自動重建上線，
手機重新整理即是最新版。不用手動上傳。

## 六、常見問題

**Q：資料會不會存到雲端？**
A：田區、紀錄、契約都存在**你手機瀏覽器本機（localStorage）**，不上傳。
換手機或清快取前，記得用「工具 → 資料備份」下載 JSON 保存。

**Q：Netlify 免費額度夠嗎？**
A：個人單場使用綽綽有餘（函式每月 12.5萬次呼叫、100GB 流量）。行情一天查幾次遠遠用不完。

**Q：可以改用 Cloudflare Pages 嗎？**
A：可以，但函式寫法（Pages Functions）與目錄慣例不同，需另外調整；先用 Netlify 最單純。

**Q：市場行情抓到的作物名對不上？**
A：AMIS 用大分類名。App 已內建對照（牛番茄→番茄、彩椒→甜椒、玉女番茄→小番茄…）。
若某作物查無資料，可先在比價頁選最接近的大分類名測試。
