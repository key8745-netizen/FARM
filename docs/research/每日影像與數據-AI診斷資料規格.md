# 每日影像與數據 — AI 診斷的客觀資料規格

> 整理日期：2026-07-22
> 目的：定義「每天要拍什麼、記什麼」，讓 AI（顧問）能做出**有依據、可比較、可動態調整**的判斷，
> 而非憑一張模糊照片猜測。
> 對象：信義鄉高冷地溫室・介質栽培・牛番茄與彩椒。

## 核心觀念（先懂這 4 點，資料才有價值）

1. **趨勢 > 單點**：同一條件連續記錄，看「變化」最有診斷力（EC 一路升？生長點越來越細？）。單獨一天的數字意義有限。
2. **標準化才客觀**：照片要固定「光線／距離／角度／背景」，量測要固定「時間／位置／方法」，不同天才能比。
3. **進排液差最關鍵**（介質栽培獨有）：給液 vs 排液的 EC/pH 差、排液率，直接反映根圈實況，比單看給液更準。
4. **生長點與新葉最靈敏**：植株最先反映營養與逆境的部位，是每日必拍必看的重點。

## 一、影像類（照片）— 拍法對了才客觀

### 拍攝標準（每次都一樣才可比較）
- **時間**：固定上午 9–10 點自然光（避免中午強光死白、傍晚偏黃）。
- **距離／角度**：全株固定退到能拍到整株；特寫固定約 20–30 cm。
- **背景**：儘量單一（可放一塊白／灰紙板當背景與白平衡參考）。
- **比例尺**：畫面放一枚硬幣或小尺 → 我能估葉片/果實**實際大小**。
- **色彩參考**：放一張白紙或色卡 → 校正白平衡，我判斷**葉色/果色**才準。
- **對焦清楚、不要逆光**。

### 每日建議拍的部位（由最有價值排序）
| 優先 | 拍什麼 | 能判斷 |
|---|---|---|
| ★★★ | **生長點＋新葉**（頂端） | 營養狀態、徒長/停長、缺素初期、病毒、蟲害初期 |
| ★★★ | **葉片正反面特寫** | 缺素（部位）、病斑、粉蝨/薊馬/葉蟎、白粉/露菌 |
| ★★★ | **黏板**（黃/藍板） | 蟲口密度（配合 IPM 門檻） |
| ★★☆ | **全株**（整株長勢） | 徒長/矮化、節間、整體均勢、萎凋 |
| ★★☆ | **果實**（含果肩、果頂） | 著色、裂果、尻腐、日燒、大小、成熟度 |
| ★☆☆ | **花／花序** | 落花、著果率、授粉狀況 |
| ★☆☆ | **介質表面／排液** | 鹽霜、藻類、青苔、乾濕、排液濁度 |

> 一張全景看不出病蟲；**特寫＋比例尺＋色卡**才是有用的照片。

## 二、數據類（可量測）— 最客觀，最好每天/每週固定

### 環境（每日）
- 溫度：**日高溫 / 夜低溫**（高冷地日夜溫差大，對著色與糖度關鍵）
- 濕度（相對濕度 %）、是否結露
- 光照（晴/陰/雨；有光量計更好）
- CO₂（若有量測）

### 根圈 / 介質（介質栽培的命脈）
- **給液 EC / pH**（配方水）
- **排液 EC / pH**（滴盤或排液口）→ **給排液差**最有價值
- **排液率**（排液量 ÷ 給液量，一般抓 20–30%）
- 介質含水率（手感或感測器）
- 每日**給液量、次數、每次時長**

### 植株生長量（每週固定測 2–3 株代表株）
- 株高、**節間長**（徒長判斷）
- **生長點到第一開花序的距離**（營養/生殖平衡的黃金指標）
- 莖徑（生長點下方莖粗）
- 每週新增葉數、目前開花節位、著果數
- 單果重、單株果數

### 產出
- 每次**採收量**與**分級**（特/優/次）→ 已可由 App 記錄

### 投入
- **施肥**：肥料種類、用量、配方 EC
- **用藥**：對象、藥劑、稀釋倍數、安全採收期（已由 App 記錄）

## 三、觀察類（半客觀）— 描述要「具體、含位置與程度」

不要只寫「葉子黃」，要寫「**下位葉**由葉緣開始黃化，約 3 片，生長點正常」——位置與程度才有診斷力。

- 生長點：強壯/細弱、顏色、是否停頓
- 葉色與**黃化位置**（上位新葉？下位老葉？→ 指向不同缺素）
- **萎凋時段**（早上就萎？中午才萎？→ 根傷 vs 蒸散過大）
- 病蟲徵狀：種類、**位置**、**數量/範圍**（幾片/幾株/整區）
- 果實異常：裂/尻腐/日燒/畸形的比例
- 當日天氣、當日農事作業（摘葉、疏果、換液…）

## 四、脈絡資料（背景，設定一次即可）

- 品種、定植日、目前生育階段（App 已有）
- 介質種類（椰纖／泥炭／…）、容器、灌溉方式（滴灌…）
- 目標值：各階段 EC/pH 目標（App 已內建）
- 歷史趨勢（App 紀錄累積）

## 五、最小可行「每日診斷包」（先從這開始，不用一次到位）

每天只要給我這 5 樣，我就能做有依據的判斷：
1. **生長點＋新葉特寫**（含比例尺）1 張
2. **異常部位特寫**（若有）1–2 張
3. **給液 / 排液 EC・pH**（4 個數字）
4. **日夜溫度**（2 個數字）
5. 一句**具體觀察**（部位＋程度）

有餘力再加：全株照、黏板照、株高/節間、採收分級。

## 六、系統怎麼幫（規劃）

App 可做：
- **每日拍照**：分類別（生長點/葉背/果實/黏板/全株）存縮圖，含日期與田區，並排看**時間序變化**。
- **每日數據表單**：一頁把上面數據填完（給排液 EC/pH、溫度、生長量、觀察）。
- **拍照提示**：拍照前提醒「放比例尺、固定光線」。
- **匯出 AI 診斷包**：把當日照片＋數據＋近期趨勢整理成一段可貼給顧問（我）的內容。
- （進階）接影像 AI：用 serverless 函式呼叫視覺模型，App 內直接回初步判斷（需 API 金鑰與成本，可選）。

> 隱私與儲存：照片建議存 IndexedDB（localStorage 太小），或上傳你自己的 Firebase Storage 跨裝置同步。
