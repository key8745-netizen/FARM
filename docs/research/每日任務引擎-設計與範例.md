# 每日任務引擎 — 設計與範例

> 整理日期：2026-07-20
> 目的：從定植日起，系統每天主動告訴使用者「今天要做什麼、怎麼做、幾天內完成」，做到無經驗也能照表操課。
> ⚠️ 數字為農業部/各區改良場公開資料之示範值，實際請以**台中區農業改良場**與**信義鄉農會**在地建議校正。

## 1. 運作原理

系統把每個作物的知識拆成三種「任務」，掛在時間軸上：

| 任務類型 | 觸發方式 | 範例 |
|---|---|---|
| **一次性任務**（固定天數）| 定植日 + N 天 | 定植後 7–10 天立支架 |
| **週期性任務**（重複）| 每 N 天／每週 | 每週檢查黃色黏板、每 2–3 週追肥 |
| **條件式任務**（看狀況）| 監測值達門檻 / 生育階段 | 粉蝨 >50 隻/板 才噴藥；第一穗著果才疏果 |

每天早上系統產生「今日待辦」，每項都附**怎麼做的圖文步驟**與**完成期限**；做完打勾、可拍照記錄。

## 2. 牛番茄逐日任務範例（溫室介質栽培・信義鄉高冷地）

### 🌱 定植期

| 時間 | 任務 | 怎麼做 | 期限 |
|---|---|---|---|
| Day 0（定植日）| 定植苗株 | 選健康苗、株距依規劃、澆定根水 | 當天 |
| Day 0–3 | 觀察活著率 | 每天巡查，補植死苗 | 3 天內 |
| Day 7–10 | **立支架／牽繩固定** | 以繩網或支架固定枝幹，讓新梢斜向上生長 | Day 10 前 |

### 🌿 生長期（定植後 2–6 週）

| 週期 | 任務 | 怎麼做 |
|---|---|---|
| 每 5–7 天 | **摘側芽（整枝）** | 設施內非停心型採**單幹整枝**：只留主幹，側芽長到 10 cm 就摘除，避免浪費養分 |
| 每 2–3 週 | **追肥** | 施於株旁 10–15 cm 處後培土（總量由計算引擎算好、分次） |
| 每日 | **溫度管理** | 花果期溫度勿低於 15°C 或高於 35°C，否則落花落果；高冷地冬季注意保溫 |
| 每日 | **澆水** | 開花前勿過濕；結果期充分且穩定，忌忽乾忽濕（防裂果） |
| 每週 | **掛/查黃色黏板** | 監測粉蝨；作為是否用藥的依據（見第 3 節）|

### 🌸 開花結果期

| 觸發 | 任務 | 怎麼做 |
|---|---|---|
| 開花 | **輔助授粉** | 溫室無風/蜂，需人工或震動輔助授粉，提高著果 |
| 第一穗著果後 | **疏果** | 疏果＋整枝＋摘葉，讓養分集中、避免器官競爭 |
| 下位老葉黃化 | **摘除老葉** | 改善通風、降低病害 |

### 🍅 採收期（定植後約 60 天起）

| 任務 | 怎麼做 |
|---|---|
| **分批採收** | 適熟即採，長期作物可持續採收數週～數月 |
| **採收期防治優先非農藥** | 採收期盡量用非農藥方式；必須用藥時**嚴格遵守安全採收期** |
| **記錄產量** | 每批採收重量入帳，累積批次資料 |

## 3. 「多久噴一次藥」— 正確且安全的邏輯 ⚠️

**系統不會叫你固定幾天噴一次藥**（那會導致農藥殘留超標、違法、也傷成本）。改用綜合管理（IPM）：

```
每週巡查 + 看黃色黏板
   ↓
未達門檻 → 不噴，繼續監測
達門檻（例：粉蝨 >50 隻/板/週）
   ↓
查「植物保護資訊系統」→ 找該病蟲害的合法登記用藥
   ↓
依標示劑量施用 → 記錄用藥日
   ↓
系統自動鎖定「安全採收期」→ 未過期前禁止採收並提醒
```

IPM 其他做法（系統會提示）：用健康種苗、嫁接抗病砧木、清除病株、掛黏板監測、必要時用天敵（如小黑花椿象防粉蝨/薊馬/蚜蟲）。

**合法用藥查詢**：植物保護資訊系統 https://otserv2.acri.gov.tw/PPM/
**番茄病蟲害用藥摘要**（高雄場）：https://www.kdais.gov.tw/upload/kdais/files/news/4964/tomato.pdf

## 4. 每日推播長什麼樣（示意）

> 📅 **10/12（定植後第 8 天）今日待辦**
> 1. ⏰ **立支架牽繩**（期限：今天）— 點看步驟圖
> 2. 🔁 巡查活著率、補植死苗
> 3. 🌡️ 今日夜溫預測 13°C，偏低 → 注意保溫（點看做法）
> 4. 💧 今日澆水：苗期少量，保持濕潤勿積水
>
> ✅ 全部完成後打卡；拍照可留紀錄

## 4.5 系統可串接的即時資料
- **天氣/低溫預警**：中央氣象署 API（高冷地冬季低溫、颱風）→ 觸發保溫/防風任務
- **合法用藥**：植保資訊系統 → 條件式用藥任務
- **行情**：農產品交易行情 API → 採收期出貨建議
