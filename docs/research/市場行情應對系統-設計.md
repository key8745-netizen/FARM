# 市場行情應對系統 — 設計文件

> 整理日期：2026-07-20
> 目的：建立一套「懂市場、會決策」的系統，用官方行情資料協助使用者決定**何時種、何時採、何時出、送哪個市場、賣什麼價**，把價格波動的風險降到最低、收入拉到最高。
> 對象：信義鄉高冷地・牛番茄與彩椒（可擴充）。

## 1. 資料來源（官方即時行情）

### 農產品交易行情 API
- 開放平台：https://data.gov.tw/dataset/8066 、 https://data.moa.gov.tw/open_detail.aspx?id=037
- 查詢站：https://amis.afa.gov.tw/
- 涵蓋範圍：**蔬菜 15–17 個批發市場、水果 11–13 個市場、花卉 5 個市場**。
- 更新頻率：每日約 AM 10:30。
- 未帶查詢日期時，預設回傳近四天資料。

### API 主要欄位
| 欄位 | 說明 |
|---|---|
| 交易日期 | 該筆行情日期 |
| 種類代碼 / 作物代號 / 作物名稱 | 定位作物（需對照代碼表）|
| 市場代號 / 市場名稱 | 定位批發市場 |
| **上價 / 中價 / 下價**（元/公斤）| 高中低分級成交價 |
| **平均價**（元/公斤）| 當日均價 |
| **交易量**（公斤）| 當日成交量（反映供給）|

> API 說明文件內含「作物代碼表、市場代碼表、種類代碼表」，串接時需先對照取得牛番茄、彩椒的作物代號與目標市場代號。

## 2. 系統五大功能

### 功能一：每日行情儀表板
- 顯示牛番茄、彩椒在**各市場的今日均價、上/中/下價、交易量**。
- 與昨日、上週、去年同期比較（↑↓ 標示）。
- 一眼看出「現在價格好不好、量多不多」。

### 功能二：價格走勢與季節分析
- 拉**歷史資料**畫出月/週價格曲線，辨識**淡旺季規律**。
- 典型規律：**盛產期（大量採收）價格最低；產季空窗、庫存見底時價格上揚**。
- 幫你找出「哪個月採收單價最好」。

### 功能三：計畫性生產回推（種植決策）
- 由「最佳採收月份」→ 回推「最佳定植日」（採收 = 定植 + 生育天數）。
- 結合信義鄉**高冷地產期優勢**：平地夏季缺貨時你有貨 = 卡好價格。
- 建議**分批定植**（見採收策略文件），避開單一崩盤期、分散價格風險。

### 功能四：出貨決策助手（採收期每日）
- 每日拉行情 → 判斷「**現在出 vs 再等**」：
  - 價格處在近期高點、走勢轉弱 → 建議出貨
  - 價格低、量能萎縮、可能反彈 → 建議可留（但須考量作物耐儲性與品質）
- **多市場比價**：比較各批發市場均價與運費，建議送哪個市場最划算。
- **分級策略**：依上/中/下價差，建議分級包裝（特/優/良）以提高整體收益。

### 功能五：行情警示推播
- 設定關注作物與門檻，觸發推播：
  - 「牛番茄均價突破 X 元，適合出貨」
  - 「近期價格連續下跌，注意調整出貨節奏」
  - 「明年同期為傳統高價期，可規劃增產」

## 3. 決策邏輯（把行情變成行動）

```
歷史行情 → 找最佳採收月 → 回推定植日 → 分批種植（分散風險）
                                    ↓
採收期每日行情 → 現在出/等？ + 送哪市場 + 分級 → 最大化收益
                                    ↓
              價格警示推播 → 提醒關鍵時機
```

## 4. 風險與注意（誠實提醒）

- **行情會受天氣、颱風、全國供給劇烈影響**，歷史規律僅供參考，非保證。
- 作物**耐儲性有限**，「等高點」不能無限等，須權衡品質與損耗。
- **量價關係**：大家都看好某作物 → 一起搶種 → 盛產崩盤。分批＋差異化（產期、品質、履歷）才是抗跌根本。
- 系統定位為**決策輔助**，最終出貨判斷仍需結合現場品質與通路合約。

## 5. 與其他模組的整合

| 模組 | 連動 |
|---|---|
| 採收與計劃性生產策略 | 行情決定分批梯次與產期 |
| 每日任務引擎 | 採收期插入「今日行情＋出貨建議」任務 |
| 產銷履歷 | 有履歷的產品可進更高價通路，行情外的溢價 |
| 契作 | 契作固定價 vs 現貨行情，系統比較何者有利 |
