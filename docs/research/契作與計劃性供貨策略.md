# 契作（契約農業）實務與系統應用分析

> 整理日期：2026-07-22
> 目的：研究台灣契作實務運作，分析哪些值得做進「農務小幫手」系統。
> 對象：信義鄉高冷地溫室・牛番茄與彩椒（無經驗新農，走安全農業/產銷履歷）。

## 1. 什麼是契作

生產前，買方（企業/通路/合作社/餐廳/加工廠）與農民**簽約**，約定在一定期間內、以一定價格、生產一定**種類、數量、規格**的農產品。

- **農民得到**：價格與銷路穩定，不必煩惱菜要賣去哪、不必天天看行情爛價。
- **買方得到**：來源與品質穩定、可追溯，能規劃自己的銷售。
- **代價**：契作價常比市價**低約 20%**，但買方**整批全收、以量取勝**，農民仍有穩定利潤。

## 2. 契作的三種型態（權利義務差很大）

| 型態 | 運作 | 定價權 | 風險 |
|---|---|---|---|
| **保價收購** | 約定**保底價**，市價高於保底常可就高 | 有保底保障 | 買方通常要求量與規格達標 |
| **行銷契作** | 買方下單載明規格/價格/數量，**生產由農民全權負責** | 契約約定價 | 產量、品質未達標的風險在農民 |
| **包銷契約** | 農民生產、**所有權移轉給買方**，由買方定售價 | **農民無定價權** | 若契約未約定風險分擔，損失可能歸農民 |

> ⚠️ 簽「包銷」前務必看清楚：所有權何時移轉、價格怎麼算、天災減產與品質不符的風險由誰負擔。與寄售/代銷（賣不掉退回、農民仍有貨權）完全不同。

## 3. 契約應載明的事項（簽約檢查清單）

- **作物與品項**、**契作期間**（起訖）
- **每週/每期供應量**（貨量）與**總量**
- **價格計算方式**（保底價？隨市浮動？固定價？）
- **分級規格標準**（大小、重量、色澤、糖度、外觀瑕疵容許）
- **交貨方式與地點**（毛重/實重、包裝規格、配送責任）
- **驗收與拒收標準**（不符規格如何處理：折價？退回？）
- **貨款請款與支付方式、付款週期**
- **違約與不可抗力條款**（天災減產、買方拒收的處理）
- **認證要求**（產銷履歷 TAP／有機／GAP：多數通路契作會要求）

## 4. 農民端的契作法則：量力而為 ＋ 風險管理

1. **量力而為**：承諾量**不要押滿全部產能**。建議只用穩定產能的 **70–80%** 去簽契作，保留 20–30% 緩衝，應對天災、病蟲害減產與品質落差，避免供不足而**違約**。
2. **分散風險**：不把全部產量壓在單一買方/單一契約；契作＋自售/市場並行。
3. **規格先行**：簽約前確認自己種得出契約要求的**分級規格**（溫室彩椒常要求色澤均勻、單果重、無日燒/裂果）。
4. **成本試算**：契作價 − 生產成本仍要有利潤才簽；低價保底的意義是**保住爛價期不虧本**。
5. **誠信履約紀錄**：穩定達標、品質一致，才能續約與談更好價格——這正是產銷履歷派上用場之處。

## 5. 契作 vs 走市場：什麼時候選哪個

信義鄉高冷地的關鍵優勢是**夏季（平地颱風/盛產爛價的空窗）能供貨 → 高價**。契作策略應與此配合：

| 情境 | 建議 |
|---|---|
| **爛價期**（平地盛產、市價 < 成本） | **走契作保底**最安全，避免血本無歸 |
| **旺季高價**（夏季颱風空窗、市價遠高於契作價） | 保留部分產能**走市場**賺價差；契作只供最低承諾量 |
| **新手第一年** | 產量不穩，**契作比例宜保守**（先求穩定銷路與現金流） |
| **通路要求履歷/有機** | 用系統的**產銷履歷紀錄簿**直接對接，成為談約籌碼 |

## 6. 通路型態參考（台灣常見契作買方）

- **連鎖超市**（如全聯與有機農場契作，不經中盤、每週定量、規模化平價）
- **有機/連鎖有機店**（要求第三方認證：有機、產銷履歷）
- **果菜生產合作社**（整合農民對接大客戶，如美生菜契作）
- **餐廳/團膳/加工廠**（固定規格、穩定週供）
- **電商/宅配品牌**（溫室小番茄、彩色番茄常見自有品牌契作）

## 7. 分析：哪些值得做進系統 ✅

以下依「對信義鄉新農實際幫助 × 系統可實作性」排序：

### 🟢 高價值、建議優先做
1. **契約登錄模組**：記錄每張契約（買方、作物、規格、計價方式、保底價、每週供應量、期間、認證要求、付款週期）。→ 集中管理、簽約檢查清單內建。
2. **供貨缺口/餘量追蹤**：契約要求每週 X kg，系統從**採收紀錄**累計實際供貨，算出**達標率與缺口**，提醒是否會違約或有餘量可走市場。
3. **契作 vs 市場決策提示**：把契約**保底價**與**當月市場行情/月價指數**並列，提示「這批走契作或留市場」。串接既有行情分析。
4. **量力而為上限建議**：依田區總產能估算，建議契作承諾**不超過 70–80%**，其餘留緩衝。

### 🟡 中價值、可後續做
5. **分級規格紀錄**：採收登錄時可分**等級**記錄數量（特級/優級/次級），對接契約分級與計價。
6. **履約與付款追蹤**：每期交貨 vs 承諾、貨款請款/收款狀態，累計履約率。
7. **契約到期提醒**：合約期間結束前提醒續約洽談。

### ⚪ 低優先（先不做）
- 複雜的合約文本產生器、電子簽章（超出手機小工具範圍，且法律風險宜找專業）。

## 8. 建議的系統落地（本次實作）

先做 🟢 高價值四項的核心：**「契作管理」新分頁**，含
- 契約登錄（買方/作物/保底價/週供量/期間/規格/認證/付款）＋簽約檢查清單。
- 每張契約的**本期供貨進度**：從對應田區採收紀錄累計 → 達標率、缺口/餘量。
- **契作 vs 市場**：保底價 對 該作物當月價指數 → 綠燈（契作划算，市場更爛）/ 紅燈（市場更高，考慮留量走市場）。
- **量力而為提示**：承諾量對比田區估計產能。
